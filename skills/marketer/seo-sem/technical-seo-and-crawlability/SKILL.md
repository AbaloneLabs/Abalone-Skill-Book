# Technical SEO And Crawlability

Content and links cannot overcome a site that search engines cannot crawl, render, or index. Technical SEO is the foundation that makes a site accessible to search engines: it governs crawlability, indexability, site architecture, page speed, and structured data. Without it, even the best content remains invisible. The judgment problem is that technical SEO issues are often invisible to content teams, silent in their harm, and complex to diagnose, and they can suppress rankings site-wide. The most common failure is investing in content and link building while technical problems, broken redirects, blocked sections, slow pages, crawl waste, prevent the site from performing. The skill is auditing and maintaining the technical foundation that lets content and authority translate into rankings.

Use this skill before a technical SEO audit, before diagnosing indexing or ranking problems, before a site migration, or when content is not ranking despite quality and links. The goal is to prevent the agent from neglecting the technical foundation, from letting migrations break SEO silently, or from treating technical SEO as a one-time fix rather than ongoing maintenance.

## Core Rules

### [ ] Ensure Crawlability And Indexability First

Before any other optimization, search engines must be able to crawl and index the site's important pages. Robots directives, crawl budget, JavaScript rendering, and site access determine whether pages are even in the game. Crawlability and indexability are prerequisites, not optimizations.

- [ ] Audit robots.txt and meta robots directives for unintended blocking.
- [ ] Ensure important pages are crawlable and indexable.
- [ ] Manage crawl budget so crawlers reach important pages.
- [ ] Verify JavaScript-rendered content is accessible to search engines.

### [ ] Maintain A Logical Site Architecture

Site architecture determines how authority flows and how users and crawlers navigate. A flat, logical structure with clear hierarchy helps crawlers find pages and helps users navigate. Deep, tangled, or orphaned architecture buries pages and dilutes authority.

- [ ] Design a shallow, logical hierarchy with clear categories.
- [ ] Ensure every important page is reachable within a few clicks.
- [ ] Avoid orphan pages with no internal links.
- [ ] Use internal linking to distribute authority to priority pages.

### [ ] Manage Redirects And URL Structure Carefully

Redirects and URL structure affect both crawlability and user experience. Redirect chains and loops waste crawl budget and dilute authority; inconsistent URL structures create duplicate content. Redirects must be clean, purposeful, and maintained.

- [ ] Use 301 redirects for permanent moves and avoid chains.
- [ ] Fix redirect loops and broken links promptly.
- [ ] Maintain consistent, clean URL structures.
- [ ] Update internal links to point directly to final URLs.

### [ ] Optimize Page Experience And Core Web Vitals

Page experience, especially Core Web Vitals measuring loading, interactivity, and visual stability, is a ranking factor and a user experience necessity. Slow, janky pages rank worse and convert worse. Page experience must be measured and optimized.

- [ ] Measure Core Web Vitals and field page experience data.
- [ ] Optimize loading, interactivity, and visual stability.
- [ ] Reduce render-blocking resources and large payloads.
- [ ] Prioritize mobile page experience.

### [ ] Implement Structured Data Correctly

Structured data (schema markup) helps search engines understand content and enables rich results that increase visibility and click-through. Implemented correctly, it enhances presence in the SERP; implemented incorrectly, it can trigger penalties. Schema must be accurate and compliant.

- [ ] Implement relevant schema for the content type, e.g., product, article, FAQ.
- [ ] Validate markup with search engine tools.
- [ ] Ensure schema accurately reflects page content.
- [ ] Avoid manipulative schema that risks penalty.

### [ ] Eliminate Duplicate Content And Canonical Issues

Duplicate content, from URL parameters, print versions, or syndication, splits signals and confuses the algorithm about which version to rank. Canonical tags consolidate signals to the preferred version. Managing duplication protects ranking clarity.

- [ ] Identify duplicate content from parameters, versions, and syndication.
- [ ] Use canonical tags to consolidate to the preferred version.
- [ ] Standardize URL parameters that create duplicates.
- [ ] Monitor for accidental duplication from CMS or staging.

### [ ] Plan Migrations To Preserve SEO Equity

Site migrations, redesigns, platform changes, domain moves, are the most common cause of catastrophic SEO loss. Without careful planning, migrations break redirects, lose URLs, and destroy accumulated equity. Migration SEO must be planned before, not after, the move.

- [ ] Map all old URLs to new URLs with 301 redirects before launch.
- [ ] Audit the new site for crawlability and indexability before launch.
- [ ] Monitor indexing and rankings closely after migration.
- [ ] Fix migration-caused issues immediately.

### [ ] Manage International And Multi-Region SEO

For sites serving multiple regions or languages, international SEO uses hreflang and regional targeting to ensure the right version reaches the right audience. Misconfigured international signals send users to the wrong version and split signals. International SEO must be deliberately configured.

- [ ] Implement hreflang correctly for language and regional variants.
- [ ] Use appropriate URL structures for international versions.
- [ ] Ensure each version targets the right region in search console.
- [ ] Audit for hreflang errors regularly.

### [ ] Monitor Technical Health Continuously

Technical SEO is not a one-time audit; sites degrade as content is added, code changes, and errors accumulate. Continuous monitoring through crawl reports, index status, and error alerts catches issues before they suppress rankings. Set-and-forget technical SEO invites silent decline.

- [ ] Run regular crawls to detect new errors.
- [ ] Monitor index status and coverage reports.
- [ ] Set up alerts for crawl errors and indexation drops.
- [ ] Treat technical SEO as ongoing maintenance.

### [ ] Align Technical SEO With Content And Authority

Technical SEO is necessary but not sufficient; it works in concert with content quality and link authority. The three must be aligned: a crawlable site with poor content will not rank, nor will great content on an uncrawlable site. Coordinate technical, content, and authority efforts.

- [ ] Ensure technical fixes support content and authority strategies.
- [ ] Prioritize technical work on pages with content and authority potential.
- [ ] Communicate technical constraints to content and link teams.
- [ ] Measure the combined effect on rankings and traffic.

## Common Traps

### [ ] Content Before Crawlability

Investing in content the search engines cannot crawl or index.

### [ ] Migration Without SEO Planning

Redesigns or platform moves that destroy accumulated equity.

### [ ] Crawl Waste

Redirect chains, duplicate URLs, and low-value pages consuming crawl budget.

### [ ] Deep Or Orphaned Architecture

Pages buried or unlinked, never found by crawlers or users.

### [ ] Core Web Vitals Neglect

Slow, unstable pages that rank and convert poorly.

### [ ] Incorrect Structured Data

Schema that is wrong or manipulative, risking penalty.

### [ ] Duplicate Content Confusion

Multiple versions splitting signals and confusing ranking.

### [ ] One-Time Audit Mentality

Treating technical SEO as fixed rather than continuously maintained.

## Self-Check

- [ ] Are important pages crawlable and indexable, with no unintended blocking?
- [ ] Is the site architecture shallow, logical, and free of orphan pages?
- [ ] Are redirects clean, purposeful, and free of chains and loops?
- [ ] Are Core Web Vitals measured and optimized for loading, interactivity, and stability?
- [ ] Is structured data implemented accurately and validated?
- [ ] Is duplicate content consolidated through canonical tags and parameter management?
- [ ] Are site migrations planned with full redirect mapping and post-launch monitoring?
- [ ] Is international SEO configured with correct hreflang and regional targeting?
- [ ] Is technical health monitored continuously, not audited once?
- [ ] Are technical, content, and authority efforts coordinated for combined effect?
