# Cross Document Consistency

A single document can be internally consistent and still fail when it sits beside related documents that contradict it. Cross-document consistency is the discipline of ensuring that a set of related works, a document series, a manuscript and its front and back matter, a report and its summary, or chapters by different contributors, align in terminology, style, formatting, voice, and factual content. Inconsistency across documents is more damaging than inconsistency within one, because readers encounter the contradictions as evidence of disorganization or unreliability. Editors who edit each document in isolation produce a set where each piece is clean but the whole is incoherent.

Use this skill when editing a set of related documents, a series, a multi-author work, or any project where multiple texts must cohere. It covers aligning terminology and style across documents, checking factual and numerical consistency, harmonizing voice among contributors, and managing the cross-document style sheet. The editor's job is to make the set read as one coherent work.

## Core Rules

### Establish A Shared Style Sheet Across The Set

Cross-document consistency begins with a shared standard. If each document follows its own style sheet, the documents will diverge. Establish one style sheet that governs the entire set, recording decisions on terminology, spelling, capitalization, hyphenation, numbers, and formatting that all documents must follow.

Create the cross-document style sheet early, before individual documents diverge, and distribute it to all contributors and editors. Record every decision that recurs across documents: how key terms are spelled and capitalized, how numbers and dates are formatted, which abbreviations are used, how names and titles are treated. As new decisions arise during editing, add them and communicate updates. The shared style sheet is the single source of truth that prevents documents from drifting apart. Without it, consistency across documents is impossible.

### Align Terminology Across All Documents

Terminology is the most visible cross-document consistency issue. If one document calls it "customer" and another calls it "client," if one uses "log in" and another "login," if one capitalizes a term and another lowercases it, readers notice the contradiction and lose confidence. Align terminology across the entire set.

Compile a glossary of key terms from the shared style sheet and verify each document uses them consistently. Check not just the preferred term but its spelling, capitalization, and hyphenation. Watch for synonyms that should be unified: if the set uses "website" in one document, it should not use "web site" or "Website" in another. Check that defined terms are used with their defined meanings throughout. Terminology alignment requires a deliberate cross-document pass because each document may be internally consistent while contradicting the others.

### Check Factual And Numerical Consistency Across Documents

Facts and numbers that appear in multiple documents must match. A statistic in a summary that differs from the statistic in the full report, a name spelled differently in two chapters, a date that shifts between documents, these contradictions undermine credibility. Cross-check factual and numerical content across the set.

Identify facts, statistics, names, dates, and figures that appear in more than one document and verify they are consistent. Pay special attention to summaries, abstracts, and front matter that restate content from the main documents; these are common sources of drift when the main document is revised but the summary is not. Check that definitions, classifications, and frameworks are applied consistently. When a fact is updated in one document, update it everywhere. Factual inconsistency across documents reads as error, not variation.

### Harmonize Voice Among Multiple Contributors

Multi-author works face a voice consistency challenge: each contributor writes in their own voice, and without harmonization, the work reads as an anthology rather than a coherent whole. While preserving each contributor's legitimate distinctiveness, harmonize the elements of voice that should be shared.

Identify the voice elements that should be consistent across contributors: register and formality, use of first or third person, tense conventions, heading style and structure, and approach to the reader. Provide contributors with guidelines on these shared elements before drafting, and edit for alignment during revision. Distinguish harmonization, aligning shared elements, from homogenization, erasing each contributor's distinctive strengths. The goal is a set that reads as coherent while respecting legitimate authorial differences. Where contributors' voices diverge sharply in ways that disrupt coherence, recommend editorial intervention to align the disruptive elements.

### Align Formatting And Structure Across Documents

Formatting consistency across documents signals professionalism and aids navigation. If documents in a set use different heading styles, numbering systems, bullet formats, or layouts, readers experience the set as disjointed. Align formatting and structure according to shared templates.

Establish templates for headings, lists, tables, captions, and document structure, and apply them across all documents. Check that heading hierarchies are used consistently, that numbered and bulleted lists follow the same conventions, and that document sections appear in a consistent order where applicable. Formatting alignment is especially important for document series, chapters in a book, or reports in a portfolio, where readers move from one document to the next and expect consistent navigation. Inconsistent formatting forces readers to re-orient in each document.

### Manage Revisions To Maintain Cross-Document Consistency

Cross-document consistency is not established once; it must be maintained as documents are revised. A change in one document may require updates in others, and without a process to track and propagate changes, consistency degrades over revision cycles. Manage revisions with cross-document consistency in mind.

When a document is revised, identify any changes that affect other documents: updated statistics, changed terminology, revised definitions, modified recommendations. Flag these changes and ensure they are propagated to all affected documents. Maintain a change log that records revisions with cross-document impact. Schedule cross-document consistency checks after major revision rounds, not just at the end. Consistency is maintained through process, not just through a final check.

### Conduct A Dedicated Cross-Document Consistency Pass

Because cross-document inconsistencies are invisible when editing documents individually, a dedicated pass that compares documents against each other is necessary. This pass looks for contradictions that no single-document edit would catch. Build it into the workflow.

After individual documents are edited, conduct a pass that reads across documents for consistency: terminology, facts, voice, and formatting. Use the shared style sheet and glossary as references. This pass often catches issues that survived individual editing because each document was internally consistent. Treat cross-document consistency as a distinct editing task with its own checklist, not as something that will happen automatically.

## Common Traps

### Editing Documents In Isolation

Each document may be clean while the set contradicts itself. Edit with the set in mind.

### Terminology Drift Across Documents

Synonyms and variant forms accumulate without a shared glossary. Align terminology deliberately.

### Summaries And Front Matter Drifting From Main Content

Revisions to main documents often miss the summary. Cross-check restated content.

### Failing To Harmonize Multi-Author Voice

Contributors' divergent voices disrupt coherence. Align shared elements while preserving distinction.

### Inconsistent Formatting Across The Set

Different templates and styles make the set feel disjointed. Apply shared formatting.

### Not Propagating Revisions Across Documents

A change in one document may require updates elsewhere. Track and propagate changes.

### Skipping A Dedicated Cross-Document Pass

Single-document edits miss cross-document contradictions. Conduct a comparative pass.

## Self-Check

Before treating cross-document consistency as complete, verify:

- A shared style sheet governs the entire set, distributed to all contributors and editors.
- Terminology has been aligned across all documents, with a glossary of key terms verified.
- Factual and numerical content appearing in multiple documents has been cross-checked for consistency.
- Summaries, abstracts, and front matter have been verified against the main documents they restate.
- Voice has been harmonized among contributors on shared elements while preserving legitimate distinction.
- Formatting and structure follow shared templates across all documents.
- A process exists to propagate revisions across affected documents.
- A dedicated cross-document consistency pass has compared documents against each other.
- The set reads as coherent, with no contradictions a reader moving between documents would notice.
- Consistency has been maintained through revision cycles, not just established once.
