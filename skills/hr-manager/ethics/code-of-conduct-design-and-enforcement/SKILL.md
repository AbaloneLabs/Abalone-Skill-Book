# Code Of Conduct Design And Enforcement

A code of conduct is the organization's statement of the behaviors it expects, prohibits, and values. A well-designed code guides behavior, supports enforcement, and protects the organization; a poorly designed one is aspirational decoration that is selectively enforced, creating cynicism and legal exposure. The agent must understand that a code must be practical enough to apply, clear enough to enforce, aligned with how the organization actually operates, and consistently applied, and that a code that exists on paper but is not enforced, or is enforced selectively, is worse than no code at all because it signals that the rules do not matter. The code is a living document that must reflect the organization's real standards and be backed by real consequences.

Use this skill when authoring, revising, or enforcing a code of conduct. The goal is to make the agent create a code that is practical, enforceable, aligned with values, and consistently applied.

## Core Rules

### Ground The Code In Real Organizational Standards

A code must reflect the behaviors the organization actually expects and enforces, not aspirational ideals it cannot or will not uphold.

- Base the code on the organization's real values and operational realities, not on generic templates.
- Address the specific risks and behaviors relevant to the organization's industry, size, and culture.
- Ensure the organization is willing and able to enforce every provision; do not include provisions that will be ignored.
- Involve stakeholders, including leadership, legal, and employees, to ensure the code is grounded and owned.
- Avoid copying another organization's code wholesale, which produces a document disconnected from the organization's reality.

### Write Clearly And Specifically

A code that is vague cannot guide behavior or support enforcement. Clarity and specificity are essential.

- Use plain language, not legalese, so employees understand the expectations.
- Be specific about prohibited conduct, with examples, rather than vague prohibitions such as "be ethical."
- Define key terms, such as harassment, conflict of interest, and confidentiality, so expectations are clear.
- Organize the code logically, with sections for key topics such as workplace behavior, conflicts, data, and compliance.
- Avoid both over-specification, which creates a rigid rulebook that cannot adapt, and over-generalization, which provides no guidance.

### Cover The Key Risk Areas

A comprehensive code addresses the behaviors that create the most risk. Ensure key areas are covered.

- Workplace behavior: respect, anti-harassment, anti-discrimination, anti-bullying. See the employee-relations skills for detailed guidance.
- Conflicts of interest: disclosure, outside employment, gifts, nepotism. See the conflict of interest skill for detailed guidance.
- Data and confidentiality: protection of company and employee data, confidentiality obligations. See the data-privacy skills for detailed guidance.
- Compliance: legal and regulatory obligations relevant to the organization.
- Use of company resources: appropriate use of systems, equipment, and time.
- Reporting: how to raise concerns and the protection from retaliation. See the ethics reporting skill for detailed guidance.

### Connect The Code To Reporting And Enforcement

A code without reporting mechanisms and enforcement is aspirational. Connect it to real consequences.

- Reference the reporting channels through which concerns can be raised, as detailed in the ethics reporting skill.
- State that violations will result in disciplinary action, up to and including termination.
- State that retaliation against reporters is itself a violation.
- Ensure the enforcement process, including investigation, is defined and connected to the code. See the workplace misconduct investigation skill for detailed guidance.

### Apply The Code Consistently

Consistent application is the test of a code's credibility. Selective enforcement destroys it.

- Apply the code to all employees, regardless of role, seniority, or performance.
- Avoid the pattern of enforcing the code against lower-level employees while tolerating violations by leaders or high performers, which signals that the code is a tool of control, not a standard.
- Document enforcement decisions and their rationale, to support consistency and defend against claims of selective enforcement.
- Train managers and leaders on the code and their responsibility to uphold it, addressing the tendency to protect high performers.

### Review And Update Regularly

A code must evolve with the organization, its risks, and its legal environment.

- Review the code at least annually, and after significant organizational changes, legal developments, or major incidents.
- Update provisions that are unclear, unenforceable, or no longer relevant.
- Incorporate lessons learned from enforcement, such as gaps revealed by investigations.
- Communicate updates to employees and provide refreshed training where expectations change.

### Communicate And Train, Not Just Publish

A code that is published but not communicated is ineffective. Employees must know, understand, and internalize it.

- Provide the code at onboarding and require acknowledgment.
- Communicate the code regularly, not just annually, through training, reminders, and leadership messaging.
- Train employees on the code's key provisions, with examples and scenarios, not just a reading requirement.
- Train managers specifically on their role in upholding and enforcing the code.
- Use real situations, appropriately anonymized, to illustrate the code in action.

### Ensure Leadership Models The Code

The code's credibility depends on whether leaders uphold it. Leadership behavior is the most powerful communication of the code's meaning.

- Hold leaders to the code without exception, addressing the tendency to excuse leaders whose violations are overlooked due to their position.
- Address leader violations firmly and visibly, within appropriate confidentiality, to signal that the code applies to all.
- Expect leaders to model the code and to reinforce it with their teams.

## Common Traps

### Aspirational Decoration

Publishing a code that sounds good but is not enforced, creating cynicism and legal exposure. Ensure every provision is enforced.

### Selective Enforcement

Enforcing the code against some employees but not leaders or high performers, destroying credibility. Apply consistently.

### Vague Prohibitions

Using vague language such as "be ethical" that provides no guidance and cannot support enforcement. Be specific with examples.

### Template Without Grounding

Copying a generic code disconnected from the organization's real risks and standards. Ground the code in organizational reality.

### Publish Without Communication

Publishing the code without training or ongoing communication, leaving employees unaware of expectations. Communicate and train.

### Tolerating Leader Violations

Overlooking code violations by leaders, signaling the code is a tool of control, not a standard. Hold leaders accountable.

## Self-Check

- [ ] Is the code grounded in the organization's real values, risks, and operational realities, not copied from a template?
- [ ] Is the code clear and specific, with plain language, defined terms, and examples of prohibited conduct?
- [ ] Does the code cover key risk areas, including workplace behavior, conflicts, data, compliance, resource use, and reporting?
- [ ] Is the code connected to reporting channels, enforcement process, and consequences, including anti-retaliation protection?
- [ ] Is the code applied consistently to all employees, regardless of role or seniority, with documented enforcement decisions?
- [ ] Is the code reviewed and updated regularly, incorporating lessons learned from enforcement and legal developments?
- [ ] Is the code communicated and trained on, not just published, with managers trained on their enforcement role?
- [ ] Do leaders model the code, with their violations addressed firmly to signal universal application?
