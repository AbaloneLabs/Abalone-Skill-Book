# Code Of Conduct Design And Content

A code of conduct is the central document through which an organization states the standards it expects of its people. It is not decoration. A code that reads as aspiration without rules, or as legal prohibition without guidance, will not change behavior, will not defend the organization in an enforcement action, and will not give managers a basis to act. The hardest part of designing a code is balancing three competing pressures: it must be short enough to be read, specific enough to be enforced, and broad enough to cover conduct the drafters did not anticipate.

Use this skill before drafting a new code, rewriting an existing one, harmonizing a code across business units or jurisdictions, or reviewing a code for tone, scope, and enforceability. The goal is to make the agent think about audience, authority, tradeoffs between values and rules, cultural transferability, and the evidence a code must generate to be defensible.

## Core Rules

### Anchor The Code To Risk And Authority

Do not write a code from a generic template. The code must reflect the organization's actual risk profile, business model, regulated activities, jurisdictions, and prior misconduct. A code that ignores the organization's highest-risk conduct is worse than no code, because it implies those risks were considered and dismissed.

Before drafting, gather:

- the organization's products, services, and customer types;
- jurisdictions of operation and their conduct expectations;
- regulated activities and licensing conditions;
- prior enforcement actions, investigations, audits, and hotline themes;
- the authority under which the code is issued (board, CEO, compliance committee);
- the legal and regulatory sources the code must operationalize;
- the population the code binds (employees, contractors, directors, agents, joint venture partners).

The code should state who issued it, who it applies to, and what legal or policy authority backs it. A code with no named owner and no approval authority cannot be enforced consistently.

### Decide The Balance Of Values, Rules, And Guidance

A code mixes three layers of language, and the balance between them determines whether the code works.

Values state what the organization stands for, such as integrity, respect, or customer focus. Values inspire but do not direct specific decisions. Rules state prohibitions and obligations, such as "do not offer bribes" or "disclose conflicts." Rules are enforceable but cannot anticipate every situation. Guidance gives examples, decision questions, and escalation paths that help people apply values and rules to ambiguous cases.

A code that is only values is unfalsifiable: any conduct can be argued to align with a value. A code that is only rules becomes a legal manual that employees will not read. Strong codes use values as framing, rules for the clear prohibitions, and guidance for the gray areas where most real decisions happen.

For each major topic, decide which layer the content belongs to, and do not blur them. A statement such as "we act with integrity, so never accept gifts over the threshold" mixes a value and a rule in a way that weakens both. Separate the framing from the directive.

### Write For The Audience That Must Use It

The primary audience of a code is not lawyers or regulators. It is the employee, contractor, or agent who must decide what to do under pressure. Write at a reading level that audience can handle, use concrete examples, and avoid terms of art unless they are defined.

Consider separate audiences:

- front-line employees who need to recognize and escalate;
- managers who must model, decide, and enforce;
- senior leaders whose conduct sets tone and who face higher-risk situations;
- contractors, agents, distributors, and joint venture partners who are not employees but bind the organization;
- board members whose duties and exposure differ from management;
- workers in jurisdictions whose language, legal tradition, and cultural norms differ from headquarters.

A single global code may be appropriate, but it should be supplemented with role-based guidance, translations, and local addenda where local law requires more or differs. Do not assume headquarters language transfers unchanged.

### Make The Code Globally Applicable Without False Universality

A global organization needs a code that applies everywhere, but "global" is not the same as "uniform." Local law may prohibit conduct the code permits, require conduct the code does not mention, or define concepts differently.

When designing a global code:

- set a floor of minimum standards that applies worldwide;
- state explicitly that local law that is stricter controls;
- identify topics where local variation is expected and require local procedures;
- avoid embedding one jurisdiction's legal terminology as if it were universal;
- address cultural transferability of concepts such as gifts, hospitality, conflicts, and harassment, which differ in meaning across regions;
- ensure translations preserve legal meaning, not just literal words;
- consult local legal counsel in high-risk jurisdictions before finalizing.

False universality creates a code that is technically uniform but practically unenforceable in places where it conflicts with local expectation or law. The code should be honest about where it sets a floor and where local rules add to it.

### Build In Enforceability And Decision Support

A code that cannot be enforced will not be respected. Each directive in the code should be connected to a mechanism: who approves, who investigates, what evidence is kept, what consequences apply, and where people go with questions.

Include:

- a clear statement that violations may result in discipline up to termination, and for agents and partners, up to termination of the relationship;
- a non-retaliation commitment with a defined scope;
- reporting channels (hotline, ombuds, manager, compliance, legal) and guidance on when to use each;
- a "when in doubt, ask" instruction with named contacts;
- decision questions or tests employees can apply, such as the newspaper test, the family test, or a short escalation checklist;
- the relationship between the code and more detailed policies, procedures, and laws.

Enforceability also requires that the code not promise more than the organization will deliver. A code that promises "zero tolerance" but tolerates high performers creates a credibility gap that is itself an enforcement risk.

### Sequence Content For Readability And Retention

Order matters. Employees read the beginning of a code more carefully than the end. Put the most important framing and the highest-risk topics early.

A workable sequence:

- letter or message from leadership establishing authority and tone;
- purpose, scope, and who the code binds;
- core values or principles, kept short;
- the duty to speak up and non-retaliation;
- high-risk conduct topics, ordered by risk and relevance;
- how to get help, report concerns, and raise questions;
- consequences of violation;
- acknowledgment and effective date.

Avoid burying the reporting path or the non-retaliation commitment at the end as boilerplate. Those are operational, not decorative.

### Plan For Maintenance, Translation, And Evidence

A code is a living document. It must be reviewed on a cycle, updated when law or risk changes, version-controlled, and re-acknowledged when materially changed.

Establish:

- a named owner, usually the chief compliance officer or equivalent;
- a review cycle, typically annual or biennial, plus event-driven reviews;
- a version history with effective dates and summaries of change;
- translation governance, including which languages are authoritative and how conflicts between language versions are resolved;
- an acknowledgment process with evidence of who read and agreed to which version;
- distribution controls so superseded versions do not circulate.

The code should generate evidence: who approved it, who has acknowledged it, when it was last reviewed, and what changed. In an enforcement review, this evidence is what demonstrates the code is operational rather than decorative.

## Common Traps

### Writing A Code That Is All Aspiration

A code composed only of values cannot be violated in any provable way, so it cannot be enforced and offers no defense. Values must be paired with rules and guidance.

### Copying A Peer's Code Without Risk Alignment

Borrowing structure is reasonable, but a code copied from another organization will reflect that organization's risks, not yours. The result can omit your highest-risk conduct.

### Embedding One Jurisdiction's Law As Universal

Using terminology from a single legal system, such as a specific statute name or a uniquely local concept, makes the code confusing or wrong elsewhere. Use neutral language and local addenda.

### Promising Zero Tolerance Without Consistent Enforcement

"Zero tolerance" language is dangerous if the organization ever makes case-by-case decisions, which it will. Inconsistent enforcement under a zero-tolerance code is itself a credibility and legal risk.

### Burying The Reporting Path

If employees cannot find how to ask a question or report a concern within the first few pages, the code has failed its most operational function.

### Ignoring Non-Employee Populations

Contractors, agents, distributors, and joint venture partners often cause the highest-risk conduct. If the code does not reach them, either directly or by contract, the organization is exposed.

### Treating Translation As Literal Word Substitution

Legal and cultural meaning does not survive literal translation. A code translated word-for-word can create obligations or gaps that do not exist in the original.

### Letting The Code Drift Without Review

A code that has not been reviewed in years signals a program that is not maintained. Regulators and auditors check the effective date and revision history.

## Self-Check

- Is the code tied to the organization's actual risks, business model, jurisdictions, regulated activities, and prior misconduct, rather than a generic template?
- Does the code name its issuing authority, the population it binds, and the legal or policy sources it operationalizes?
- Is there a deliberate balance of values, rules, and guidance, with each layer separated rather than blurred?
- Is the code written for the front-line audience, with examples and decision support, not only for lawyers?
- Does the code set a global floor while honestly addressing local law that is stricter and concepts that do not transfer uniformly?
- Is each directive connected to an enforcement mechanism: approval, investigation, evidence, consequence, and escalation?
- Are the reporting path and non-retaliation commitment prominent and operational, not buried as boilerplate?
- Is content sequenced so the highest-risk topics and the duty to speak up appear early?
- Are maintenance, version control, translation governance, and acknowledgment evidence established?
- Does the code avoid promises, such as zero tolerance, that the organization cannot keep consistently?
- Are contractors, agents, distributors, and joint venture partners reached by the code or by contract?
