# Changeover And Setup Discipline

Changeover is a high-risk moment in production. Materials, tooling, labels, software recipes, cleaning state, documentation, and people all change at once. Agents often focus on reducing setup time and miss the controls that prevent contamination, wrong product, first-piece failures, or safety incidents. This skill helps the agent improve changeover discipline without turning speed into uncontrolled risk.

Use this skill when reviewing setup time, line clearance, batch change, product transition, tooling swap, cleaning, first-piece approval, changeover standard work, or SMED-style improvement. The agent should distinguish useful external setup from unsafe shortcutting.

## Core Rules

### Define the changeover boundary

Clarify when the prior run ends, what must be cleared, when setup begins, when the next run is approved, and when stable production is reached. Some teams measure only machine downtime and ignore preparation, cleaning, inspection, or startup scrap.

Use a boundary that matches operational reality. If hidden work happens before or after the measured window, the metric will mislead and improvements may shift burden elsewhere.

### Protect line clearance

Line clearance prevents old materials, labels, documents, tools, settings, or work-in-progress from contaminating the next run. It is critical for regulated, high-mix, allergen, color, lot-controlled, or customer-specific production. Clearance should have owner, checklist, evidence, and approval where required.

Do not reduce changeover time by weakening clearance. Wrong label or residual material can create serious quality and compliance failures.

### Separate internal and external setup

External setup can be done while the line is running: staging tools, pre-kitting materials, preparing documents, warming equipment, checking fixtures, or confirming next work order. Internal setup requires the line to be stopped. Move work external where safe and practical.

Do not pretend work is external if it requires unsafe proximity, distraction from current production, or premature material exposure. The distinction should improve flow without creating new hazards.

### Verify tooling, settings, and recipes

Setup often fails because the wrong tool, program, fixture, label, torque, temperature, speed, or recipe is used. Confirm the required configuration and version. Where possible, use mistake-proofing, barcode checks, digital recipes, or independent verification for high-risk setups.

If the line uses temporary settings or manual overrides, document the reason, authority, and expiration. Temporary setup workarounds easily become normal practice.

### Control startup quality

The first units after changeover should be checked against defined criteria: dimension, function, label, weight, seal, appearance, calibration, test result, contamination, or customer specification. First-piece or startup approval should not be rushed because the line is waiting.

If startup scrap or rework is common, analyze the setup process. Do not normalize early-run waste as unavoidable without evidence.

### Confirm stable running before declaring success

Changeover is not complete the moment the first acceptable unit is produced. Check whether the line can hold rate, quality, and safety conditions for a reasonable startup window. Some setup errors appear only after temperature stabilizes, tooling heats, material tension changes, or operators move from supervised startup into normal rhythm.

### Coordinate people and timing

Changeover requires operators, maintenance, quality, material handlers, sanitation, supervisors, and sometimes engineering. If any role is late or unclear, downtime grows. Schedule support roles and make handoff points visible.

Avoid starting changeover when the required approver or tool owner is unavailable. Waiting after the line is down is expensive and demoralizing.

### Reduce time without hiding risk

Changeover improvement should target search time, movement, waiting, unclear instructions, missing tools, poor staging, repeated adjustments, and approval delays. It should not remove critical checks, cleaning, safety locks, or documentation.

When a time reduction is proposed, ask which work disappeared, moved, or changed. If risk controls were moved, verify they still happen and evidence remains.

### Learn from failed setups

Track setup overruns, first-piece failures, wrong materials, missing tools, cleaning failures, safety stops, and startup scrap by product, line, shift, crew, and changeover type. Patterns reveal training, standard work, scheduling, tooling, or design issues.

A changeover problem is often a process design problem, not an individual attention problem.

## Common Traps

- Measuring only machine-down time while ignoring preparation, approval, and startup stabilization.
- Reducing setup time by weakening clearance, cleaning, or first-piece checks.
- Staging materials early in ways that create mix-up, contamination, or safety risk.
- Using manual settings or recipes without verification or version control.
- Starting changeover before support roles, tools, materials, or approvers are ready.
- Treating startup scrap as normal instead of a setup quality signal.
- Declaring changeover complete after one good unit while early-run instability continues.
- Celebrating faster changeover while defects or rework rise; blaming operators for setup failures caused by poor staging, unclear instructions, or missing tools

## Self-Check

- Is the changeover boundary defined from prior run end through stable next-run production?
- Are line clearance, cleaning, material removal, label control, and documentation protected?
- Is internal setup separated from safe external preparation?
- Are tooling, recipes, software settings, fixtures, labels, and versions verified?
- Are temporary overrides documented with authority and expiration?
- Are first-piece or startup quality checks defined and respected?
- Is stable running verified before the changeover is treated as complete?
- Are operators, maintenance, quality, materials, sanitation, and supervisors coordinated?
- Does time reduction remove waste rather than required controls?
- Are setup failures reviewed by pattern across product, line, shift, and changeover type?; would the changeover record defend the decision if a wrong-product or contamination issue occurred?
