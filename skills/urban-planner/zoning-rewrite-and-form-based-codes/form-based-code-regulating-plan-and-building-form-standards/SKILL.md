# Form-Based Code Regulating Plan and Building Form Standards

The form-based code regulates the urban form — the building, the street, the public realm — rather than the use, because the form determines the character and the walkability, and the form-based code produces the predictable and the high-quality place. The judgment problem is that the regulating plan is often imprecise (the vague zones, the no calibrating plan), the building form standards are often wrong (the inappropriate heights, the wrong setbacks, the missing frontages), and the public realm is often ignored (the no street standards, the no civic spaces), and these failures produce the form-based code that does not produce the intended place. A form-based code that is drafted well (the calibrated plan, the right form standards, the integrated public realm) produces the intended place, while a code that is drafted poorly (the imprecise plan, the wrong standards, the ignored realm) fails. This skill covers the regulating plan and the building form standards decisions that determine whether the form-based code produces the intended place. It complements the broader zoning-rewrite skill by focusing on the form-based dimensions.

## Core Rules

### Calibrate the Regulating Plan to the Context

The regulating plan should be calibrated to the context, because the plan is the map that assigns the form zones, and the calibration ensures that the zones match the existing and the intended character, and the uncalibrated plan produces the wrong form. The rule is to calibrate the regulating plan, using the context analysis (the existing character, the building heights, the setbacks, the frontages), the vision (the intended character, the plan's vision), the zone assignment (the form zones matched to the context), and the boundaries (the clear and the logical zone boundaries). The context analysis should document the existing character, because the code should fit the place, and the analysis informs the calibration. The vision should define the intended character, because the code should implement the vision, and the vision guides the calibration. The zone assignment should match the form zones to the context and the vision, because the zones are the regulatory tool, and the assignment implements the intent. The boundaries should be clear and the logical, because the unclear boundaries confuse, and the clear boundaries enable the administration.

### Set the Building Form Standards for the Intended Character

The building form standards should produce the intended character, because the standards (the height, the setback, the lot coverage, the footprint) determine the building form, and the form determines the character, and the wrong standards produce the wrong character. The rule is to set the building form standards, using the height standards (the minimum and the maximum, the story and the feet), the setback standards (the front, the side, the rear, the build-to line), the lot coverage and the footprint (the maximum coverage, the building footprint), and the parking placement (the rear, the side, the interior, the no front parking). The height standards should set the minimum and the maximum, because the form-based code ensures the consistent form, and the minimum prevents the too-low and the maximum prevents the too-high. The setback standards should use the build-to line (the front setback that builds to the sidewalk), because the build-to line creates the street wall, and the street wall defines the public realm. The lot coverage and the footprint should control the bulk, because the excessive bulk overwhelms the street, and the control manages it. The parking placement should require the rear or the interior, because the front parking destroys the walkability, and the placement preserves it.

### Establish the Frontage and the Public Realm Standards

The frontage and the public realm standards should define the relationship between the building and the street, because the frontage (the porch, the gallery, the storefront, the forecourt) and the public realm (the sidewalk, the street, the plaza) create the pedestrian experience, and the standards determine the quality. The rule is to establish the frontage and the public realm standards, using the frontage types (the gallery, the arcade, the storefront, the porch, the stoop, the forecourt), the frontage requirements (the required frontage, the transparency, the entrance), the public realm standards (the sidewalk width, the street trees, the lighting, the furnishings), and the civic space standards (the plazas, the squares, the parks). The frontage types should be specified for each zone, because the frontage activates the street, and the types define the activation. The frontage requirements should mandate the active frontage, because the blank wall and the garage door deaden the street, and the active frontage enlivens it. The public realm standards should specify the sidewalk and the street trees, because the public realm is the pedestrian space, and the standards ensure the quality. The civic space standards should define the plazas and the squares, because the civic spaces are the gathering places, and the standards create them.

### Integrate the Street Standards and the Thoroughfare Plan

The street standards and the thoroughfare plan should be integrated, because the streets are the public realm, and the thoroughfare plan defines the network, and the street standards define the cross-section, and the integration produces the walkable network. The rule is to integrate the street standards, using the thoroughfare plan (the network of the streets, the connectivity, the block size), the street types (the avenue, the street, the lane, the alley), the cross-section standards (the travel lanes, the parking, the bike, the sidewalk, the trees), and the connectivity (the connected network, the small blocks, the no cul-de-sacs). The thoroughfare plan should define the network, because the network determines the connectivity, and the plan guides it. The street types should be specified for each context, because the street type determines the cross-section, and the types define it. The cross-section standards should balance the modes, because the complete street serves all the users, and the standards ensure it. The connectivity should require the connected network and the small blocks, because the connectivity enables the walkability, and the cul-de-sac and the large block destroy it.

### Transition from the Use-Based to the Form-Based Regulation

The transition from the use-based to the form-based regulation should be managed, because the transition is the significant change, and the management ensures the acceptance and the effectiveness, and the unmanaged transition produces the confusion and the resistance. The rule is to manage the transition, using the scope (the area, the districts, the full or the optional code), the conversion (the existing uses, the nonconformities, the vested rights), the administration (the review process, the staff capacity, the training), and the adoption (the political support, the stakeholder engagement, the education). The scope should define the area and the type (the full code, the optional code, the overlay), because the scope determines the impact, and the definition guides it. The conversion should address the existing uses and the nonconformities, because the form-based code regulates the form, and the existing nonconforming uses must be handled. The administration should build the staff capacity, because the form-based code requires the different skills, and the capacity enables it. The adoption should build the political support, because the form-based code is the change, and the support enables it.

## Common Traps

### Regulating Plan Uncalibrated, Wrong Zones

The regulating plan is uncalibrated (the generic zones, the no context analysis), and the wrong zones are assigned, because the calibration was not conducted. The trap is that the uncalibrated plan produces the wrong form. The mechanism is that the calibration was not conducted, and the generic zones were assigned. The false signal is that "we have a form-based code." The harm is that the zones do not match the context, the form is wrong, the character is not achieved, and the calibration that should match the zones does not.

### Building Form Standards Wrong, Character Not Achieved

The building form standards are wrong (the inappropriate heights, the wrong setbacks), and the character is not achieved, because the standards were not calibrated. The trap is that the wrong standards produce the wrong character. The mechanism is that the standards were not calibrated, and the generic standards were used. The false signal is that "we have form standards." The harm is that the buildings are the wrong form, the character is not achieved, the place is not walkable, and the standards that should produce the character do not.

### Frontage Ignored, Blank Walls and Dead Streets

The frontage is ignored (the no frontage requirements, the blank walls allowed), and the streets are dead, because the frontage standards were not established. The trap is that the ignored frontage produces the dead streets. The mechanism is that the frontage standards were not established, and the buildings turn their back to the street. The false signal is that "the buildings are built to the sidewalk." The harm is that the frontages are blank, the streets are dead, the walkability is destroyed, and the frontage that should activate the street does not.

### Street Standards Absent, Public Realm Poor

The street standards are absent (the no thoroughfare plan, the no cross-section), and the public realm is poor, because the street standards were not integrated. The trap is that the absent standards produce the poor realm. The mechanism is that the street standards were not integrated, and the streets were left to the engineering. The false signal is that "the code regulates the buildings." The harm is that the streets are auto-oriented, the public realm is poor, the walkability is destroyed, and the standards that should create the realm do not.

### Transition Unmanaged, Confusion and Resistance

The transition is unmanaged (the no conversion plan, the no staff training), and the confusion and the resistance occur, because the transition was not managed. The trap is that the unmanaged transition produces the confusion. The mechanism is that the transition was not managed, and the code was adopted. The false signal is that "the code is adopted." The harm is that the staff is unprepared, the applicants are confused, the resistance builds, and the management that should enable the transition does not.

## Self-Check

- Is the regulating plan calibrated, with the context analysis, the vision, the zone assignment, and the boundaries?
- Are the building form standards set for the character, with the height, the setback, the coverage, and the parking?
- Are the frontage and the public realm standards established, with the frontage types, the requirements, the public realm, and the civic spaces?
- Are the street standards integrated, with the thoroughfare plan, the street types, the cross-section, and the connectivity?
- Is the transition managed, with the scope, the conversion, the administration, and the adoption?
- Is the plan calibrated, or are the zones wrong?
- Are the standards right, or is the character wrong?
- Is the frontage addressed, or are the streets dead?
- Are the street standards integrated, or is the realm poor?
- Is the transition managed, or is there confusion?
