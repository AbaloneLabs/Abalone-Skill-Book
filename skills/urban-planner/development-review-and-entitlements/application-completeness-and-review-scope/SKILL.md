# Application Completeness and Review Scope

The completeness check is the gate that determines whether a development application moves forward, gets delayed, or is rejected before substantive review even begins. The judgment problem is not mechanical box-checking but deciding whether the application contains enough information for the reviewing body to make legally required findings, whether the project as described matches the entitlements being sought, and whether the review scope captures every required permit, study, and condition — because anything missed at this stage becomes a liability, a delay, or a lawsuit later. Planners commonly fail by treating completeness as a formality (rubber-stamping thin applications), by over-asking for studies that are not actually required (which invites claims of arbitrary process), or by missing a required entitlement or environmental trigger that surfaces months into review. A rigorous completeness and scoping decision protects the applicant, the agency, and the eventual approval.

## Core Rules

### Define Completeness Against the Required Findings, Not a Generic Checklist

A complete application is one that contains enough information for the decision-maker to make every finding the code requires for the entitlements sought. Work backward from the findings: if the code requires the review body to find that the project is consistent with the comprehensive plan, compatible with surrounding uses, served by adequate facilities, and free of undue environmental impact, then the application must contain the analysis and evidence to support each of those findings. A generic submittal checklist is a starting point, but the real standard is whether the specific project, on its specific site, gives the reviewer what is needed to reach the required conclusions. When in doubt, ask: "What would I need to see to write the staff report's findings for this project?" If you cannot write the findings from what is submitted, the application is not complete.

### Verify the Entitlements Match the Project as Described

A frequent and dangerous error is allowing an application to proceed under the wrong entitlement path — for example, a project that needs a conditional use permit proceeding as a site plan review, or a project that requires a variance being processed as an administrative approval. Read the project description against the code's trigger criteria carefully: does the use as proposed actually meet the definition of the use being permitted? Does the intensity, density, or form require discretionary review that the applicant has not requested? Does the project involve any element (demolition, signage, lot split, access change) that triggers a separate entitlement? Mapping every required approval at the start prevents the worst kind of delay: discovering a missing entitlement weeks before a hearing.

### Scope the Required Studies to Actual Impacts, Not Default Templates

Many codes and agencies have default study requirements (traffic study above X trips, noise study within Y feet of a freeway, geotechnical report on slopes above Z percent). Apply these triggers, but also assess whether the specific project creates impacts the templates do not capture — shadowing on a sensitive neighbor, view obstruction, loading conflicts on a narrow street, cumulative effects with other pending projects. Conversely, do not require studies that the project clearly will not need; over-scoping invites claims that the process is being used to obstruct rather than evaluate, and it wastes applicant and staff time. Tie each required study to a specific impact question it is meant to answer, and be able to defend that linkage.

### Identify Environmental Review Triggers Early

Environmental review (under state environmental policy acts, NEPA for federal actions, or local equivalents) has its own completeness and scoping logic, and its timing interacts with the entitlement process. Determine at the outset whether the project is exempt, requires a negative declaration or mitigated negative declaration, or requires a full environmental impact statement/report — and whether the environmental document must be completed before, alongside, or after the entitlement decision. Missing an environmental trigger, or sequencing it wrong, is a common source of legal challenge and project-killing delay. When the answer is unclear, resolve it in writing early, because ambiguity discovered late is far more costly.

### Coordinate Concurrent Reviews and Conditions Up Front

A development project often requires approvals from multiple departments — planning, building, public works, fire, health, environmental, historic, utilities — and the conditions each imposes must be reconciled into a coherent set, not stacked independently. Convene the relevant reviewers early, identify overlapping and potentially conflicting conditions, and resolve them before the staff report is written. A condition set that contains internal contradictions (one department requires a wider driveway, another requires a narrower one) signals an uncoordinated review and creates grounds for appeal. The lead planner's job is to synthesize conditions into a clear, consistent, enforceable set tied to specific code authority.

### Document the Completeness Determination in Writing

The completeness determination itself is a decision that can be challenged, so it must be documented: what was submitted, what was reviewed, what was found complete or incomplete, and the date the completeness clock started. If the application is deemed incomplete, provide a specific, itemized list of what is missing and the code basis for each requirement, so the applicant can cure the deficiency. Vague incompleteness determinations ("needs more information") are not actionable, invite disputes about whether the clock is running, and can be overturned on appeal for lack of specificity.

## Common Traps

### Rubber-Stamping Thin Applications to Keep the Clock Moving

Under pressure to hit review timelines, planners sometimes accept applications that are technically complete on paper but lack the substance needed to write defensible findings. The false signal is that review is "on schedule"; the harm is that the gaps surface during hearing preparation or, worse, in the findings themselves, which then cannot be supported by the record. Hold the line on substantive completeness even when it means a delay, because a delayed-but-defensible approval is always cheaper than a fast-but-vulnerable one.

### Missing a Required Entitlement or Permit

A project that quietly needs a variance, a conditional use permit, a floodplain review, or a historic approval — none of which the applicant requested — can sail through the headline entitlement only to be challenged or stalled on the missing approval. The false signal is that the primary review is going smoothly; the harm is a project that cannot actually be permitted as approved. Build a entitlement map for every project at intake and have a second reviewer check it, because the cost of a missed approval compounds the later it is discovered.

### Requiring Studies That Are Not Legally Justified

In response to neighborhood pressure or general caution, reviewers sometimes demand studies, analyses, or conditions that the code does not actually require for the project's impacts. The false signal is thoroughness; the harm is that the applicant can challenge the requirements as arbitrary, exactions beyond the project's impacts, or pretextual obstruction — and may win. Tie every required study and condition to a specific code provision and a specific project impact, and be ready to defend the nexus.

### Letting Conditions Accrue Without Reconciliation

When each reviewing department adds conditions independently, the final condition set can be contradictory, redundant, or impossible to satisfy simultaneously. The false signal is a long, comprehensive-looking conditions list; the harm is that the applicant faces unworkable or conflicting requirements, appeals, and construction-phase disputes. The lead planner must reconcile conditions into a coherent, internally consistent set before the decision is issued.

### Treating the Completeness Decision as Administrative Rather Than Substantive

Because completeness is often processed by intake staff, it can be treated as clerical rather than as a substantive legal determination. The false signal is a fast turnaround; the harm is that completeness errors — wrong entitlement, missing environmental trigger, inadequate project description — propagate through the entire review and surface as fatal defects at the hearing or in litigation. Treat completeness review as a substantive planning decision performed or checked by someone who understands the full entitlement path.

## Self-Check

- Does the application contain enough information for the decision-maker to make every finding the code requires for each entitlement sought — not just to satisfy a generic checklist?
- Have you confirmed that the entitlements requested actually match the project as described, including all secondary approvals (variances, conditional uses, environmental, historic, access)?
- Is every required study tied to a specific impact question and a specific code trigger, with none required that the project's impacts do not justify?
- Have you determined the environmental review path (exempt, negative declaration, EIS/EIR) and its correct sequencing relative to the entitlement decision?
- Have all relevant reviewing departments been coordinated, and are their conditions reconciled into a consistent, non-contradictory set tied to code authority?
- Is the completeness determination documented in writing with specific findings and, if incomplete, an itemized cure list with code citations?
- Did a qualified planner (not just intake staff) verify the entitlement map and environmental triggers for the project?
- Could you write the required findings for each entitlement from the information in the application as submitted — and if not, is the application actually complete?
- Are there any conditions or studies being imposed that you cannot tie to a specific code provision and a specific project impact?
- If this approval were challenged, would the completeness and scoping record withstand review for thoroughness, correctness, and lack of arbitrariness?
