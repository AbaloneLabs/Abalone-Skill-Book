# Brand Color And Type System

Color and type are the two systems a brand uses most often and abuses most often. They are also the two systems that, once defined poorly, create low-grade chaos in every asset that follows. The judgment problem is that choosing colors and fonts looks like a creative, taste-driven act, but a functional brand system is mostly engineering: defining a constrained palette, a coherent type scale, accessible contrast, and clear rules for use. Creators fail by choosing too many colors, by picking fonts that cannot be licensed or loaded where needed, by ignoring contrast until assets are unreadable, and by leaving the system so loosely defined that every asset improvises.

The harm is persistent. Inconsistent color and type make a brand look amateur and erode recognition across the feed. This skill helps the agent build color and type systems that are constrained, accessible, usable everywhere, and easy to follow.

## Core Rules

### Constrain The Palette To A Workable Set

More colors do not make a richer brand; they make a noisier one. A functional palette is small, with clear roles for each color, so every asset knows which color to use for what.

Build a role-based palette:

- one or two primary brand colors that carry recognition;
- a small set of secondary or accent colors for emphasis;
- neutrals for backgrounds, text, and structure;
- semantic colors for status, such as success, warning, and error, where relevant.

Define each color's role explicitly. A palette without roles becomes a box of crayons where anything goes.

### Define Exact Values In The Right Color Spaces

Brand colors drift the moment they are specified loosely. Every color needs exact values in the spaces it will be used, and those values must be the single source of truth.

Specify precisely:

- hex values for digital use;
- RGB for screens and CMYK or Pantone for print where relevant;
- the color space, such as sRGB, to avoid shifts across devices;
- tints and shades derived by rule, not by eye.

Eyeballed colors scatter across a feed. Exact, documented values are what keep a palette intact.

### Ensure Accessibility And Contrast From The Start

Color choices that ignore contrast produce assets that are unreadable for part of the audience and non-compliant with accessibility standards. Contrast is a constraint that must be designed in, not fixed later.

Check contrast for every pairing:

- text against its background meets the relevant contrast threshold;
- interactive elements are distinguishable beyond color alone;
- color is not the sole carrier of meaning, since some users cannot perceive it;
- the palette includes options that work for users with color vision deficiencies.

Designing accessibility in early is far cheaper than retrofitting it, and it expands the audience rather than excluding them.

### Choose Typefaces That Work Everywhere They Are Needed

A brand typeface must be usable in every context the brand operates in: web, video, print, social, and possibly multiple languages. Choosing a face that cannot be licensed, loaded, or rendered in a needed surface breaks the system.

Validate typeface fitness:

- confirm licensing covers all intended uses;
- verify web and app availability or suitable fallbacks;
- check support for the languages and scripts the brand needs;
- ensure the family has enough weights for a full type scale.

A beautiful face that cannot be used in video overlays or on the website is not a viable brand typeface.

### Build A Type Scale, Not A Collection Of Sizes

A type system is a small set of related sizes and weights mapped to roles, not a free choice of size per asset. A defined scale is what makes a brand feel structured and consistent.

Define the scale:

- a fixed set of sizes for headline, subhead, body, caption, and utility;
- a fixed set of weights, usually two or three;
- consistent line height and spacing per role;
- rules for how the scale applies across surfaces.

When every asset invents its own sizes, the brand looks disorganized. The scale enforces coherence by default.

### Pair Fonts With Clear Roles

If the system uses more than one typeface, each must have a defined role so they complement rather than compete. Pairing without roles produces visual conflict.

Assign roles clearly:

- a display or headline face for emphasis;
- a text or body face for readability;
- possibly a mono or accent face for specific utility, such as code or labels;
- rules for which face leads in which context.

One face should carry the brand voice; the other should support it. Two loud faces fighting is a common amateur signal.

### Document Usage Rules So The System Scales

A color and type system only works if people know how to use it. Undocumented systems decay the moment they are handed off or revisited after time.

Document at minimum:

- each color's role, value, and acceptable pairings;
- the type scale with sizes, weights, and use cases;
- what is allowed and forbidden, such as stretching type or using off-palette colors;
- examples of correct and incorrect application.

Documentation is what turns a set of choices into a maintainable system.

### Plan For Variation Without Breaking The System

A rigid system with no room to flex becomes stale; a loose system with no rules drifts. The goal is planned variation within defined boundaries.

Allow controlled variation:

- campaign or seasonal color emphasis drawn from the existing palette;
- format-driven type adjustments within the scale;
- imagery and illustration styles that shift while color and type stay fixed;
- sub-brands or series with their own constrained frame.

Planned variation keeps the brand alive; unplanned variation erodes it. The difference is whether the variation was designed into the rules.

## Common Traps

### Too Many Colors With No Roles

A large palette without assigned roles becomes noise. Constrain the set and define each color's job.

### Loose Or Eyeballed Color Values

Approximate colors scatter across assets. Specify exact values in the right color spaces.

### Ignoring Contrast Until Assets Are Unreadable

Retrofitting contrast is costly and often impossible without redesign. Design accessibility in from the start.

### Typefaces That Cannot Be Used Where Needed

A face that fails licensing, web loading, or language support breaks the system. Validate fitness across all surfaces.

### Ad Hoc Sizes Instead Of A Type Scale

Inventing sizes per asset looks disorganized. Define and reuse a fixed scale.

### Competing Typefaces With No Roles

Two loud faces without assigned roles fight each other. Give each a clear job.

### Undocumented Systems That Decay On Handoff

Without documentation, the system drifts the moment it leaves the original author. Write the rules down.

## Self-Check

Before approving a brand color and type system, verify:

- The palette is constrained to a workable set with a defined role for each color.
- Every color has exact values specified in the relevant color spaces.
- Text-to-background pairings meet accessibility contrast thresholds.
- Meaning is never carried by color alone, with alternatives for colorblind users.
- Brand typefaces are licensed and usable across web, video, print, and required languages.
- A defined type scale exists with fixed sizes, weights, and roles.
- If multiple typefaces are used, each has a clear, non-competing role.
- Usage rules, including what is forbidden, are documented for handoff.
- Planned variation is built into the rules so the brand can flex without drifting.
- The system was tested across real assets, not only in isolation on a style guide page.
