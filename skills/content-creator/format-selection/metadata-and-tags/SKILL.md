# Metadata And Tags

Metadata is the information about the content that platforms and search engines use to understand, categorize, and surface it. The judgment problem is that creators treat metadata as either irrelevant, leaving it default, or as a manipulation game, stuffing tags and keywords to game discovery. Both fail. Ignored metadata leaves content undiscoverable to audiences who would want it. Manipulated metadata mislabels content, attracts the wrong audience, and is increasingly penalized by platforms that detect spam.

The harm is mismatch and suppression. Missing or vague metadata means search and recommendation systems cannot connect the content to the right audience. Misleading metadata attracts viewers who bounce, signaling low relevance and suppressing future reach. Keyword stuffing triggers spam filters. This skill helps the agent write metadata that accurately represents content, serves both discovery and accessibility, and avoids the manipulation tactics that backfire.

## Core Rules

### Treat Metadata As Accurate Description, Not Manipulation

The purpose of metadata is to describe the content accurately so the right audience can find it. Manipulation, stuffing, and mislabeling produce short-term gains at best and long-term penalties at worst.

Write metadata to:

- describe what the content actually is about;
- use the terms the target audience uses to search;
- place the content in the correct category and context;
- help platforms understand and recommend it to the right viewers;
- support accessibility for users who rely on metadata and alt text.

Accurate metadata aligns the content with the audience that wants it. Manipulated metadata misaligns both and gets punished.

### Use The Audience's Search Language

Metadata should use the words the audience uses, not the creator's internal terminology. Discovery depends on matching how people actually search and browse.

Use audience language by:

- researching the terms the target audience searches with;
- including natural phrasing of the audience's questions;
- avoiding jargon the audience does not use;
- covering relevant synonyms and related terms honestly;
- not forcing awkward keyword phrases into readable text.

Metadata in audience language connects content to demand. Metadata in creator jargon does not.

### Write Descriptions That Serve Humans And Systems

Descriptions are read by humans deciding whether to click and by systems deciding what the content is about. Serve both.

Write descriptions by:

- leading with a clear statement of what the content offers;
- including relevant terms naturally, not stuffed;
- providing context that helps the audience decide to engage;
- adding timestamps, links, and structure where useful;
- keeping the most important information near the top.

A description stuffed with keywords repels humans. A description that ignores keywords misses search. Balance both.

### Choose Tags And Categories Accurately

Tags and categories help platforms place content. Choose them for accuracy and relevance, not for maximum volume.

Choose tags by:

- selecting tags that genuinely describe the content;
- prioritizing specific over generic tags where relevant;
- including the terms the target audience would use;
- avoiding irrelevant popular tags that mislabel the content;
- using the platform's category system correctly.

Tagging a piece with popular but irrelevant terms attracts the wrong audience, causes bounces, and can trigger spam penalties. Accuracy beats volume.

### Provide Alt Text And Accessibility Metadata

Metadata is not only for discovery. Alt text, captions, and accessibility metadata make content usable for people with disabilities and are increasingly required.

Provide accessibility by:

- writing descriptive alt text for images and graphics;
- ensuring captions are accurate and synchronized;
- including descriptive metadata for audio where supported;
- not leaving accessibility fields empty when they exist.

Accessibility metadata serves real users and often improves discovery as a side benefit. Treat it as core, not optional.

### Avoid Keyword Stuffing And Spam Patterns

Platforms detect and penalize metadata manipulation. Keyword stuffing, tag spamming, and hidden text trigger spam filters and can suppress or remove content.

Avoid spam by:

- not repeating keywords unnaturally in descriptions;
- not listing dozens of marginally relevant tags;
- not hiding keyword text in descriptions or metadata fields;
- not using deceptive categorization to reach unrelated audiences;
- keeping metadata honest and readable.

Manipulation worked in earlier eras of the web. It now triggers penalties that set the content back further than honest metadata would have.

### Match Metadata To Each Platform's System

Each platform has different metadata fields, weights, and norms. Metadata that works on one platform may be irrelevant or counterproductive on another.

Match by platform:

- use the specific fields each platform provides;
- respect character limits and display truncation;
- learn which fields each platform weights for discovery;
- adapt categorization to each platform's taxonomy;
- update metadata when platforms change their systems.

Copy-pasting the same metadata everywhere ignores platform differences and underperforms tailored metadata.

### Maintain And Update Metadata Over Time

Metadata is not set-and-forget. Content can be re-categorized, search trends shift, and platforms change. Maintain metadata to keep content discoverable.

Maintain by:

- reviewing metadata on evergreen content periodically;
- updating terms as audience language evolves;
- refreshing descriptions when content is updated;
- re-categorizing when platform taxonomies change;
- monitoring which terms actually drive discovery.

Stale metadata lets evergreen content fade. Maintenance keeps it findable.

### Measure Which Metadata Drives Discovery

Use platform analytics to learn which metadata actually connects content to audiences, and refine based on evidence.

Measure by:

- tracking which search terms bring viewers to the content;
- identifying which tags and categories correlate with reach;
- comparing metadata variations across similar pieces;
- cutting metadata that misleads or attracts the wrong audience;
- doubling down on terms that drive relevant discovery.

Measurement turns metadata from guesswork into a learnable practice.

## Common Traps

### Ignoring Metadata Entirely

Default or empty metadata leaves content undiscoverable to audiences who would want it.

### Keyword Stuffing

Unnatural keyword repetition repels humans and triggers spam penalties.

### Mislabeling For Reach

Tags and categories chosen for popularity rather than accuracy attract the wrong audience and cause bounces.

### Creator Language Over Audience Language

Metadata in internal jargon misses the terms the audience actually searches with.

### Skipping Accessibility Metadata

Empty alt text and missing captions exclude users and forgo discovery benefits.

### Copy-Pasting Across Platforms

Ignoring platform-specific metadata fields and norms underperforms tailored metadata.

### Set-And-Forget Metadata

Stale metadata lets evergreen content fade as search trends and platforms evolve.

## Self-Check

Before approving metadata for a piece, verify:

- Metadata accurately describes the content rather than manipulating discovery.
- Terms used match the audience's search language, not the creator's jargon.
- Descriptions serve both human readers and discovery systems, with key information first.
- Tags and categories are accurate and relevant, not chosen for volume.
- Alt text, captions, and accessibility metadata are provided where fields exist.
- No keyword stuffing, hidden text, or spam patterns are present.
- Metadata is tailored to each platform's fields, limits, and weighting.
- Evergreen content has a maintenance plan for refreshing metadata.
- Platform analytics are used to learn which terms drive relevant discovery.
- No misleading categorization attracts the wrong audience and causes bounces.
