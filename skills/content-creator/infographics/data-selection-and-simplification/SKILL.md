# Data Selection And Simplification

An infographic is not a dataset with pictures around it. It is an argument or a story built from a small, deliberate selection of data. The judgment problem is that simplification is necessary for communication but is also where accuracy dies. Every cut, rounding, and aggregation is a choice that can clarify or distort, and creators routinely fail in two opposite directions: cramming every number they found into the graphic until it is unreadable, or stripping so much context that the remaining figure misleads. The ethical weight is real, because audiences treat a charted number as verified fact and rarely check the source.

The harm is both intellectual and reputational. An overloaded infographic communicates nothing. An oversimplified one communicates the wrong thing confidently. This skill helps the agent select data that serves a true message and simplify it without bending it.

## Core Rules

### Start From The Message, Not The Dataset

The most common failure is to begin with a spreadsheet and try to display everything in it. Effective infographics begin with a single message the audience should understand, then select only the data that supports that message.

Define the message first:

- what is the one thing the audience should take away;
- what claim or comparison does the data need to support;
- what would the audience need to know to find this credible.

Only after the message is clear should you go to the data and choose what to include. Data that does not serve the message should be excluded, no matter how interesting.

### Select The Smallest Set That Proves The Point

Simplification is the core craft. The goal is the minimum amount of data that makes the message clear and defensible, not the maximum amount that fits on the canvas.

Apply selection discipline:

- include only the figures directly relevant to the claim;
- aggregate detail into categories when the pattern is what matters;
- remove redundant metrics that say the same thing differently;
- prefer one strong comparison over several weak ones.

Every additional number competes for attention and dilutes the message. If removing a number does not weaken the argument, remove it.

### Preserve Context That Changes The Meaning

Simplification becomes distortion the moment it removes context the audience needs to interpret the number correctly. A figure without its baseline, timeframe, or comparison can be technically true and deeply misleading.

Always retain context that affects meaning:

- the timeframe the data covers;
- the baseline or comparison point, such as versus last year or versus a competitor;
- the unit and scale, including whether axes do not start at zero;
- the sample size or population when it affects significance.

Simplify the presentation, never the context that prevents misreading.

### Round And Aggregate Without Distorting

Rounding and aggregation make numbers readable, but each must be done so the simplified figure still represents reality. Aggressive rounding can erase real differences, and careless aggregation can hide important variation within groups.

Round and aggregate safely by:

- rounding only to the precision the message requires;
- keeping rounding consistent across compared figures;
- documenting how categories were combined;
- checking that aggregation does not reverse a trend visible in the raw data.

If a rounded or aggregated version of the data tells a different story than the raw data, the simplification has gone too far.

### Distinguish Correlation, Causation, And Comparison

Audiences read two numbers placed together as related, and a chart as proof of causation. The creator is responsible for making the relationship explicit rather than implying more than the data supports.

Be explicit about the relationship:

- label comparisons as comparisons, not causes;
- avoid implying causation from correlational data;
- show the comparison basis clearly, such as before versus after or group A versus group B;
- include caveats when a strong pattern could have alternative explanations.

A graphic that lets the audience infer causation the data does not prove is misleading by omission.

### Choose The Right Comparison Frame

A number means little without a frame of reference. The choice of comparison frame can make the same figure look impressive or alarming, and that choice is an editorial one the creator owns.

Pick the frame deliberately:

- compare to a relevant baseline, average, or prior period;
- use per-capita or rate-based comparisons rather than raw totals where scale misleads;
- avoid cherry-picked comparison points chosen only to flatter the message;
- disclose the frame so the audience can judge it.

The most common manipulation in infographics is not a fake number but a real number in a misleading frame.

### Match Data Density To The Audience And Surface

The right amount of data depends on who will read it and where. A social scroll tolerates far less than a report or a long-form article.

Calibrate density:

- social and thumb-stop graphics, one or two figures and a single comparison;
- blog or article graphics, a small set with supporting context;
- reports and dashboards, denser data with full sourcing.

Pushing report-level density into a social graphic guarantees it will not be read. Respecting the surface is part of honest simplification.

### Keep The Source Visible And Credible

An infographic without a visible source is an assertion. Audiences and critics increasingly check claims, and unsourced graphics are easy to dismiss or debunk.

Attach source information:

- name the source clearly, not just a vague institution;
- include the date the data was collected or published;
- link or reference the specific dataset where possible;
- note any methodology that affects interpretation.

Sourcing is not decoration; it is what separates an infographic from an opinion with clip art.

## Common Traps

### Displaying The Whole Dataset

Cramming every number into the graphic makes it unreadable and buries the message. Select for the message, not for completeness.

### Stripping Context That Changes Meaning

Removing timeframe, baseline, or scale can turn a true number into a misleading one. Simplify presentation, not context.

### Implying Causation From Correlation

Placing two trend lines together invites a causal reading the data may not support. Make the relationship explicit.

### Cherry-Picked Comparison Frames

Choosing a flattering baseline or time window manipulates without falsifying. Pick frames that represent reality.

### Rounding That Erases Real Differences

Over-rounding can collapse meaningful gaps into apparent equality. Round only to the precision the message needs.

### Unsourced Or Vaguely Sourced Figures

Graphics without clear, dated sources are easy to dismiss and hard to trust. Attach specific sourcing.

### Wrong Density For The Surface

Report-level density on a social graphic guarantees it is ignored. Match data load to where it will be read.

## Self-Check

Before approving data selection and simplification, verify:

- The message was defined before any data was selected, and only supporting data was included.
- The smallest defensible set of figures is shown, with no number that does not serve the message.
- Context that changes meaning, such as timeframe, baseline, scale, and sample size, is retained.
- Rounding and aggregation were checked against the raw data and do not reverse the real trend.
- The relationship shown, comparison or correlation, is labeled explicitly and does not imply unsupported causation.
- The comparison frame was chosen to represent reality, not to flatter the message.
- Data density matches the surface where the infographic will appear.
- A specific, dated source is attached and verifiable.
- Removing any single remaining figure would either weaken the message or be safe to do.
- A skeptical reader checking the source would find the graphic accurately represents it.
