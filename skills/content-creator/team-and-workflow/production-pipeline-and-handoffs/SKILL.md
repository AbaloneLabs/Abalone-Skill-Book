# Production Pipeline And Handoffs

A production pipeline is a system, and systems fail at their seams. The judgment problem is that creators design pipelines around the creative stages they enjoy, ideation, filming, editing, while leaving the transfers between stages implicit, and those implicit handoffs are exactly where work falls through, gets duplicated, or stalls. Most production failures are not failures of any single stage but failures of the handoff between stages, where one person assumes the work moved and the other assumes it was received. A good pipeline treats handoffs as explicit, documented transfers with clear inputs, outputs, and acceptance criteria, and it makes the whole workflow visible so nothing depends on memory or proximity.

The harm is dropped tasks, duplicated work, and bottleneck formation. Implicit handoffs produce pieces that disappear between people and deadlines that slip silently. Unclear quality gates mean output drifts as more people touch it. A pipeline that lives in one person's head is fragile to absence and turnover. This skill helps the agent map production stages end to end, design handoffs as explicit transfers, set quality gates that catch problems early, make the workflow visible to the whole team, and protect the creator's deep work from being fragmented by constant coordination demands.

## Core Rules

### Map The Pipeline End To End, Including Invisible Stages

A pipeline only works when every stage is visible, including the ones creators forget, like research, approvals, and distribution. Map the whole flow.

Map end to end by:

- listing every stage from ideation through publishing and distribution;
- including administrative stages like approvals, rights, and compliance;
- identifying the owner and output of each stage;
- surfacing stages that currently happen implicitly or invisibly;
- treating the map as the shared model of how work moves.

Stages that are not mapped are stages where work gets lost.

### Design Handoffs As Explicit Transfers

Most pipeline failures happen at handoffs. Treat each handoff as an explicit transfer with defined inputs, outputs, and acceptance, not an assumption.

Design handoffs by:

- defining what a completed handoff looks like for each stage;
- specifying the format, assets, and information required at each transfer;
- setting acceptance criteria so the receiver knows it is complete;
- using shared tools so handoff status is visible to everyone;
- closing the loop so nothing is assumed handed off that was not received.

Implicit handoffs produce dropped tasks; explicit handoffs make the pipeline reliable.

### Set Quality Gates At Meaningful Checkpoints

Quality drifts as more people touch content, and quality gates catch problems before they compound. Place gates where they catch issues cheaply.

Set gates by:

- placing review checkpoints before work becomes expensive to change;
- defining what each gate checks against, using documented standards;
- making gates checkable, not just felt, so quality is consistent;
- avoiding so many gates that the pipeline stalls;
- feeding gate feedback back into standards so quality improves over time.

A gate at the right point prevents a small problem from becoming a published one.

### Make The Workflow Visible To The Whole Team

A pipeline that lives in one person's head is fragile. Make status, ownership, and dependencies visible to everyone on the team.

Make visible by:

- using shared project management for status and ownership;
- maintaining a single source of truth for deadlines and dependencies;
- holding regular brief check-ins on status and blockers;
- documenting processes so they survive team changes;
- making quality standards and templates accessible to all.

Invisible workflows depend on memory; visible workflows scale and survive turnover.

### Protect The Creator's Deep Work From Coordination Drag

A pipeline that constantly interrupts the creator for approvals and decisions fragments their deep work and recreates the bottleneck. Structure the flow to protect focus.

Protect deep work by:

- batching approvals and reviews rather than dripping them in;
- setting decision thresholds so minor choices do not reach the creator;
- scheduling handoffs and reviews to align with the creator's focus patterns;
- delegating coordination so the creator is not the workflow hub;
- recognizing that fragmented attention costs more than delayed approvals.

A pipeline optimized for constant creator input optimizes the bottleneck it should remove.

### Build In Buffer For Delays And Dependencies

Content production is full of dependencies and unpredictable delays, and a pipeline with no buffer breaks when any stage slips. Build buffer deliberately.

Build buffer by:

- scheduling slack between dependent stages;
- accounting for review and revision cycles in timelines;
- not planning every stage back to back with no margin;
- identifying the critical path and protecting it;
- treating buffer as a feature, not wasted time.

A pipeline with no buffer turns every small delay into a missed deadline.

## Common Traps

### Implicit Handoffs

Assuming work moved between people without explicit transfer produces dropped tasks and duplicated effort.

### Invisible Stages

Forgetting to map administrative and approval stages causes work to stall where no one owns the transfer.

### Too Few Or Too Many Quality Gates

No gates let quality drift; excessive gates stall the pipeline and frustrate the team.

### Workflow Living In One Person's Head

A pipeline that depends on memory and proximity breaks under absence, scaling, or turnover.

### Constant Creator Interruptions

A flow that routes every approval and decision to the creator fragments their deep work and recreates the bottleneck.

### No Buffer Between Stages

A pipeline with zero slack turns every small delay into a cascading missed deadline.

### Unclear Acceptance Criteria

Handoffs without defined completion criteria leave receivers unsure whether the work is truly done.

## Self-Check

- The pipeline is mapped end to end, including administrative and approval stages.
- Every handoff is an explicit transfer with defined inputs, outputs, and acceptance criteria.
- Quality gates are placed at meaningful checkpoints with documented, checkable standards.
- Workflow status, ownership, and dependencies are visible to the whole team.
- The creator's deep work is protected from coordination drag and constant interruptions.
- Buffer is built in between dependent stages to absorb delays.
- The critical path is identified and protected.
- Handoff status is tracked in shared tools, not assumed in conversation.
- The number of quality gates balances catching problems against pipeline speed.
- The pipeline would survive a key person's temporary absence without collapsing.
