# Onboarding And Activation

Onboarding is the bridge between acquisition and retention. It is the sequence of steps that takes a new user from arrival to their first experience of the product's core value, the moment called activation. If onboarding fails, users leave before they ever discover why the product is worth keeping, and all the acquisition effort and spend is wasted. Good onboarding is ruthlessly focused on reaching that first value moment as fast and reliably as possible; poor onboarding collects information, showcases features, or delays value, and loses users who never see why they should stay.

Use this skill before answering broad questions such as "how do I design onboarding", "how do I activate users", "why do users sign up but not use the product", or "how should an entrepreneur onboard new customers". The goal is to prevent the agent from designing onboarding that serves the venture's needs rather than the user's path to value.

## Core Rules

### Define Activation As Reaching The Core Value

Activation is not sign-up, not first login, not a tour. It is the moment the user first experiences the core value of the product, the outcome that makes the product worth using. Define activation concretely for the specific product: the action or state that means the user has reached value. Everything in onboarding should move toward that moment.

### Remove Every Step That Delays Value

Audit each onboarding step and ask whether it moves the user toward activation. Steps that collect information, explain features, or serve the venture's analytics rather than the user's progress delay the value moment. Remove or defer anything that does not advance the user to activation. The fastest path to value is the best onboarding.

### Front-Load The Value, Defer The Rest

Show value before asking for commitment. Let users experience the core value with minimal friction, registration, configuration, payment, before asking them to invest in setup, profiles, or paid plans. Front-loading value builds the motivation that makes later commitment feel worthwhile; demanding commitment first loses users who have no reason to commit.

### Guide Users Explicitly To The Value Moment

Users do not discover the core value by accident. Guide them explicitly, through clear steps, prompts, and defaults, to the action that produces value. Ambiguity in onboarding loses users who would have valued the product if they had found their way. Make the path to value obvious and hard to miss.

### Measure Activation Rate And Time-To-Value

Measure what percentage of new users reach activation, and how long it takes them. These metrics reveal whether onboarding works: a low activation rate means users are lost before value; a long time-to-value means friction delays the moment that would retain them. Optimize both, because each affects retention.

### Segment Activation By Source And User Type

Activation rates differ by acquisition source and user type. Users from different channels, segments, or use cases may need different onboarding to reach value. Segment activation metrics to find which sources or types struggle, and tailor onboarding where generic onboarding loses specific groups.

### Reduce Cognitive Load In Onboarding

Each choice, field, and explanation in onboarding adds cognitive load that increases abandonment. Reduce choices to the essential, pre-fill where possible, and explain only what the user needs to act now. Cognitive load is the enemy of activation; simplicity is its ally.

### Test Onboarding Changes Against Activation

Treat onboarding as a series of experiments. Each change, fewer steps, clearer guidance, different ordering, should be tested against activation rate and time-to-value, not against opinion. Let evidence settle onboarding debates, and keep the changes that measurably improve the path to value.

### Connect Onboarding To The Core Loop

Onboarding should not be a separate experience from the product; it should lead directly into the core value loop the user will repeat. An onboarding that ends at a different place than where ongoing use begins creates a gap users must bridge alone. Connect onboarding seamlessly to the loop that drives retention.

### Acknowledge Personal Risk And Professional Advice

Onboarding may involve data collection, payment, and claims with legal and regulatory implications that require professional advice. This skill does not replace that counsel.

## Common Traps

### Onboarding Serving The Venture, Not The User

Collecting information and showcasing features delays the value moment.

### Activation Defined As Sign-Up

Counting registration as activation hides users lost before value.

### Demanding Commitment Before Value

Asking for setup or payment before showing worth loses unmotivated users.

### Ambiguous Path To Value

Users who cannot find the core value leave, even if they would have valued it.

### Cognitive Overload

Each extra choice and field increases abandonment before activation.

### Onboarding Disconnected From The Core Loop

A separate onboarding that ends away from ongoing use creates a gap.

## Self-Check

- [ ] Activation is defined concretely as reaching the core value, not as sign-up.
- [ ] Every onboarding step is audited for whether it advances the user toward activation.
- [ ] Value is front-loaded before commitment is asked.
- [ ] Users are guided explicitly to the value moment, not left to discover it.
- [ ] Activation rate and time-to-value are measured and optimized.
- [ ] Activation is segmented by source and user type to find struggling groups.
- [ ] Cognitive load is minimized through fewer choices and pre-filled defaults.
- [ ] Onboarding changes are tested against activation metrics, not opinion.
- [ ] Onboarding connects seamlessly to the core value loop.
