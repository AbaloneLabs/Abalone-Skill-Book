# Pivot Types And Tradeoffs

Pivots are not all alike. A customer-segment pivot preserves the product but changes who it serves; a problem pivot preserves the customer but changes what need is addressed; a platform pivot changes the technology foundation; a business-model pivot changes how value is captured; a technology pivot changes how the solution is built. Each type preserves different assets and sacrifices different work, and each carries different risk and validation requirements. Choosing the pivot type is the discipline of matching the change to the evidence and to what the venture should preserve.

Use this skill before answering broad questions such as "what kind of pivot should we make", "what are the types of pivots", "should we change our customer or our product", "what does each pivot preserve", or "how should an entrepreneur choose a pivot type". The goal is to prevent the agent from treating all pivots as equivalent and from choosing a pivot scope that does not fit the evidence or the venture's assets.

## Core Rules

### Match The Pivot Type To The Evidence

Different evidence warrants different pivot types. Evidence that the product works but for a different customer suggests a customer-segment pivot; evidence that the customer is right but the need is different suggests a problem pivot; evidence that the model does not capture value suggests a business-model pivot. Match the type to what the evidence actually shows.

### Understand What Each Pivot Preserves

Each pivot type preserves different assets: a customer-segment pivot preserves the product; a problem pivot preserves customer relationships; a business-model pivot preserves the product and customer but changes monetization; a technology pivot preserves the problem and customer but changes the build. Choose a pivot that preserves what is working, to give the new direction a running start.

### Understand What Each Pivot Sacrifices

Conversely, each pivot sacrifices work: a customer-segment pivot sacrifices the current customer base and positioning; a problem pivot sacrifices the current solution; a business-model pivot sacrifices the current revenue mechanics. Understand the sacrifice, because it represents sunk work and risk. A pivot that sacrifices too much is closer to starting over.

### Customer-Segment Pivot: Same Product, New Customer

When the product works but for a different customer than originally targeted, a customer-segment pivot preserves the product and redirects go-to-market. This pivot is common when an unintended segment shows stronger demand. Its risk lies in rebuilding positioning and channels for the new segment.

### Problem Pivot: Same Customer, New Need

When the customer relationship is strong but the original problem is not worth solving, a problem pivot preserves the customer and addresses a different, more valuable need. This pivot leverages trusted access. Its risk lies in building a new solution and validating the new problem's worth.

### Business-Model Pivot: Same Product And Customer, New Monetization

When the product and customer are right but the economics do not work, a business-model pivot, such as subscription to usage, or direct to channel, changes how value is captured. This pivot preserves the core offering. Its risk lies in disrupting revenue and revalidating unit economics.

### Platform Or Technology Pivot: New Foundation

When the underlying technology or platform is wrong, a platform or technology pivot rebuilds the foundation while preserving the problem and customer. This pivot is expensive and slow but sometimes necessary. Its risk lies in the rebuild cost and the transition execution.

### Choose The Smallest Pivot That Addresses The Evidence

Larger pivot types sacrifice more and carry more risk. Choose the smallest pivot type that addresses the evidence, because over-pivoting discards valuable work and raises risk. Let the evidence define the pivot scope.

### Validate The New Direction For The Chosen Type

Each pivot type requires validating different things: a customer-segment pivot needs demand validation in the new segment; a problem pivot needs problem-worth validation; a business-model pivot needs unit-economics validation. Validate the specific assumptions the chosen pivot type resets.

### Consider The Asset And Capability Fit

The pivot type should fit the venture's assets and capabilities: a technology pivot requires build capability; a customer-segment pivot requires go-to-market capability. Choose a pivot type the venture can execute, because a strategically right pivot the team cannot execute still fails.

### Acknowledge Personal Risk And Strategic Judgment

Pivot type choices carry significant consequences and operate under uncertainty. Seek advisor, board, and team input, and validate with evidence before committing. This skill provides framing, not certainty.

## Common Traps

### Pivot Type Mismatched To Evidence

Changing the wrong element when the evidence points to a different pivot type.

### Over-Pivoting

Choosing a larger pivot than the evidence warrants sacrifices too much.

### Ignoring What Is Preserved

A pivot that discards working assets is closer to starting over.

### Validating The Wrong Assumptions

Each pivot type resets different assumptions; validating the wrong ones wastes effort.

### Pivot The Team Cannot Execute

A strategically right pivot fails if it exceeds the venture's capabilities.

### Treating All Pivots As Equivalent

Different pivot types have different risk, validation, and asset implications.

## Self-Check

- [ ] The pivot type is matched to what the evidence actually shows.
- [ ] What the chosen pivot preserves is understood and leveraged.
- [ ] What the chosen pivot sacrifices is understood and accepted.
- [ ] Customer-segment, problem, business-model, and technology pivots are distinguished.
- [ ] The smallest pivot type that addresses the evidence is chosen.
- [ ] The specific assumptions the pivot resets are validated.
- [ ] The pivot type fits the venture's assets and capabilities.
- [ ] The pivot is not larger than the evidence warrants.
- [ ] The right assumptions, not the wrong ones, are validated for the pivot type.
- [ ] The choice draws on advisor, board, and team input under acknowledged uncertainty.
